** TODO [#A] operations.org — Tagesgeschaeft & Schuljahres-Rhythmus
Das eigentliche Administrators Handbook. Kein Aufbau, sondern Betrieb.

Inhalt:
- Schuljahresbeginn: 60-rein-60-raus
  - students.csv vorbereiten (Format, Pflichtfelder)
  - provision-students.sh durchlaufen lassen
  - Initial-Passwoerter ausgeben/drucken (must-change on first login)
  - Verifizieren dass alle Logins funktionieren
- Schuljahresende:
  - Graduierte archivieren (ZFS-Snapshot vom Home, Account disable)
  - Daten an Schueler auf USB ausgeben falls gewuenscht
  - Klassenmaterial-Pruefung (was wird weitergetragen, was geloescht)
- Software-Updates:
  - NixOS Channel Bump (=nixpkgs-unstable= -> neuer Release)
  - =nixos-rebuild switch --flake .#mkuu1=
  - Client-Closure neu bauen, Pfad in hosts/web/configuration.nix
    aktualisieren, web01 switchen
  - Clients reboot triggern (per SSH oder physisch)
- Rollback-Verfahren wenn Rebuild kaputt:
  - =nixos-rebuild switch --rollback= auf mkuu1
  - Bei Client: alten clientSystem-Store-Pfad zurueck in web config
- User-Verwaltung im Alltag:
  - Passwort eines Schuelers zuruecksetzen
  - Neuen Schueler nach Schuljahresbeginn nachtragen
  - Account temporaer sperren (Disziplinarmassnahme)
- Backup-Rotation:
  - Wann welche der zwei USB-Platten anschliessen
  - Wie verifizieren dass das letzte Backup vollstaendig ist
  - Wo die Platten zwischen den Rotationen lagern (Brandschutz)

** TODO [#A] runbook.org — Symptom -> Diagnose -> Fix
Wenn's brennt. Pro Symptom kurz und konkret.

Faelle die abgedeckt sein sollten:
- "Schueler kann sich nicht anmelden"
  - Account in lldap pruefen, sssd-Cache invalidieren, sssctl user-checks
- "Mein Home-Verzeichnis ist weg / leer"
  - NFS-Mount check, ZFS dataset, ggf. aus Snapshot wiederherstellen
- "Client bootet nicht"
  - PXE-Stadium (kein DHCP / kein iPXE / kein NFS / Hang in systemd)
  - Wo die Bootmeldung typischerweise stehenbleibt
- "Container startet nicht / startet immer wieder neu"
  - =machinectl status=, journalctl, NixOS-Activation-Fehler
- "Internet ist langsam"
  - MikroTik QoS, Airtel-Tarif-Status, evtl. Schueler-Tab mit Stream
- "Drucker / USB-Geraet wird nicht erkannt"
- "ZFS scrub findet Fehler"
  - Lesefehler vs. Pruefsummen-Fehler unterscheiden, Reaktion je nach Schwere
- "UPS piept dauerhaft"
- "Bildschirm flackert" (FIFO-Underrun, GPU)

Jeder Eintrag: ein Code-Block mit den 2-3 Diagnose-Kommandos +
ein Code-Block mit dem Fix. Kurz, kein Roman.

Was darf der Lehrer in Tansania allein tun, wenn Axel in DE nicht
erreichbar ist?

Inhalt:
- Erlaubt:
  - Hard-Reset eines haengenden Clients (5 Sek Powerknopf)
  - Server neu starten ueber USV-Knopf (wenn vorher graceful shutdown
    auf dem Server selber)
  - Backup-USB-Platte tauschen nach festem Plan
  - Maus/Tastatur/Kabel austauschen
  - Drucker-Papier nachfuellen, Patronen wechseln
- Nicht anfassen:
  - SSH-Terminals
  - MikroTik Web-UI
  - ZFS / nixos-rebuild
  - lldap Web-UI
- Notfall: Was tun bei Stromausfall mit defekter USV?
- Eskalations-Kontakte:
  - Axel (WhatsApp, Email)
  - Lokaler Shop (InnoSykes Moshi) — fuer Hardware-Defekte
- Telefonliste in einer Mappe an der Wand neben dem Server
